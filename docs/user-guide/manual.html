1第一次使用

這個系統可以幫你管理「進貨、賣東西、看庫存、看每月報表」這些事。打開瀏覽器（Chrome、Edge 都可以）就能用，不用裝任何東西。

怎麼打開系統

  1. 打開瀏覽器
    建議用 Chrome 或 Microsoft Edge。
  2. 輸入網址
    在網址列輸入 http://localhost:5173 然後按 Enter。
  3. 輸入帳號密碼
    你會看到一個登入畫面。輸入老闆／管理員給你的帳號和密碼，按 登入。
  4. 進到首頁
    登入成功就會看到一個有「本月銷貨額」「本月進貨額」「低庫存品項」等卡片的畫面。
小提示 忘記密碼？找系統管理員（admin）幫你重設，他在「使用者管理」可以幫你改新密碼。

3角色和你能做什麼

每個人有一個角色，角色決定你能看什麼、做什麼。共四種：

  • admin 系統管理員 全部都能做，只有他能新增 / 刪除帳號。應收應付都能收/付款。
  • manager 主管 可以管商品、開進貨單、開銷貨單、確認入庫、確認出貨；可以幫客戶記應收收款、付供應商應付付款。
  • sales 業務 可以開銷貨單、確認出貨、記應收收款（客戶付錢進來）；商品、進貨、付供應商只能看。
  • warehouse 倉管 可以「確認入庫」進貨單；其他大部分只能看，不能收/付款。

誰能在「應收」「應付」按收款 / 付款？

  • 應收收款（客戶付錢給你）：adminmanagersales
  • 應付付款（你付錢給供應商）：adminmanager（業務不能付供應商的錢）
  • 查看清單、明細、帳齡：四種角色都可以看。
注意 看不到的按鈕表示你的角色不能做那件事，這不是 bug。如果你覺得應該要能做，找你的主管或系統管理員調整。
4整體流程：進貨、銷貨、應收、應付怎麼串起來

系統在做兩條平行的事 — 一條是東西的流向（進來、賣出去），一條是錢的流向（誰欠你錢、你欠誰錢）。兩條線在「確認」那一刻會自動接起來。

一張圖看懂

進貨這條線（你付錢給供應商）

  1. 建立進貨單（草稿） 跟供應商談好的訂單。此時還沒進庫存、也還不欠錢。
  2. 確認入庫（已入庫） 貨真的到了倉庫。系統做三件事：庫存↑、商品成本更新、自動產生一張應付帳款。
  3. 應付帳款（未付） 編號 AP-…。到期日 = 入庫日 + 供應商的「付款條件天數」。
  4. 記一筆付款 到「應付帳款」找到那張，按「記錄付款」。可以分次付。
  5. 付完了（已付清） 付清後狀態自動變成已付清；中間還沒付清是 部分付款。

銷貨這條線（客戶付錢給你）

  1. 建立銷貨單（草稿） 客戶下單。此時還沒扣庫存、客戶也還沒欠你錢。
  2. 確認出貨（已出貨） 真的把貨交出去了。系統做兩件事：庫存↓、自動產生一張應收帳款。
  3. 應收帳款（未收） 編號 AR-…。到期日 = 出貨日 + 客戶的「付款條件天數」。
  4. 記一筆收款 客戶付錢來了，到「應收帳款」找到那張，按「記錄收款」。可分次。
  5. 收完了（已收齊） 收齊後狀態自動變已收齊；中間是 部分收款。

關鍵觀念（這幾點搞懂就會用了）

  • 「確認」那一刻是關鍵。在進貨單按「確認入庫」、銷貨單按「確認出貨」之前，不會產生應付 / 應收，也不會動庫存。確認之後就鎖住了。
  • 應收 / 應付不能自己手動建。它們是進貨單 / 銷貨單「確認」時系統自動產生的，每張進貨單對到一張應付、每張銷貨單對到一張應收。
  • 進貨單 / 銷貨單沒辦法直接刪除已確認的單。如果發現應收應付有問題，要從原始的進貨 / 銷貨單去釐清，或建反向單沖掉（找主管）。
  • 「付款條件天數」是預設值。每個客戶 / 供應商身上都有一個（預設 30 天），可以個別調。改了之後只會影響「之後」新產生的應收應付，不會回頭改舊的。
  • 稅系統固定算 5% 營業稅。建單時可以選「含稅」或「未稅」— 含稅表示你填的金額已經含稅，系統會反推未稅金額；未稅就是把稅加上去。應收應付上會看到三欄：未稅、稅額、含稅總額。

一個具體例子

舉例 5/01 客戶「永豐行」下單買 100 瓶可樂 = $5,250（含稅）。
5/02 你按「確認出貨」 → 系統扣 100 瓶庫存，自動產生應收 AR-20260502-0001，金額 $5,250，到期日 = 5/02 + 30 天 = 6/01。
5/15 客戶先付 $3,000 → 你在應收明細按「記錄收款」填 3,000，狀態變成 部分收款，未收餘額 $2,250。
6/05 客戶才付剩下 $2,250 → 再記一筆，狀態變 已收齊。
因為 6/01 已過、6/05 才付清，這段期間在「帳齡分析」會被算進「逾期 1-30 天」桶（直到付清為止）。

狀態快速對照

進貨單 / 銷貨單
草稿 → 已入庫 / 已出貨 （或 已取消 只能在草稿時）
應收 / 應付
未付/未收 → 部分 → 付清/收齊（已過到期日會多一個 逾期 紅標）
記住 只有草稿才能改。一旦確認入庫或出貨就鎖住了 — 因為這時候已經影響庫存跟帳款，亂改會錯亂。應收 / 應付本身的金額也不能改，只能透過「記錄收/付款」或「作廢付款」來調整。
5商品分類 adminmanager

把商品分成大類別，例如「飲料」「零食」「日用品」。先建分類，再建商品時選分類，比較好整理。

新增一個分類

  1. 左側選單點「商品分類」
  2. 右上角點 + 新增
  3. 輸入分類名稱
    例如「飲料」。
  4. 填說明（可不填）
  5. 按 儲存

編輯或刪除

在分類清單裡，每一列右邊都有「鉛筆」（編輯）跟「垃圾桶」（刪除）。

注意 刪掉分類後，原本屬於這個分類的商品「分類欄」會變空白，但商品本身還在。
6商品 adminmanager

這裡管你賣的東西。每個商品要有一個「商品編號（SKU）」和名稱。庫存量在這裡不能直接改 — 它會由進貨、銷貨自動算。

新增商品

  1. 左側選單點「商品管理」
  2. 右上角點 + 新增
  3. 填基本資料
    商品編號（例：A001）、名稱（例：可口可樂 330ml）、單位（個／瓶／包）。
  4. 選分類
    如果之前沒建分類，可以先去第 5 章建。
  5. 填售價跟成本
    售價是賣給客戶的價錢。成本不用填得很精準 — 之後每次進貨確認入庫時系統會自動更新。
  6. 低庫存閾值（選填）
    例如填 10，當庫存 ≤ 10 時系統會在庫存查詢顯示「低庫存」紅字提醒。填 0 表示不要提醒。
  7. 按 儲存
不要這樣做 不要為了調整庫存而修改商品。要調整庫存就建一張「進貨單」或「銷貨單」，這樣才有紀錄可查。

找商品

在最上面的搜尋框輸入商品編號、名稱、或條碼，會即時篩選。也可以用「分類」「狀態」下拉縮小範圍。

7供應商 adminmanager

記錄你從誰那邊進貨。建立進貨單之前要先有供應商。

新增供應商

  1. 左側選單點「供應商」
  2. 右上角點 + 新增
  3. 填供應商名稱
    這欄是必填，名稱不能跟既有的重複。
  4. 聯絡人、電話、Email、地址、統編、備註
    能填就填，方便日後對帳。
  5. 付款條件（天）
    這欄很重要。它決定了之後每張「進貨單入庫」自動產生的應付帳款「到期日」要算幾天 — 例如填 30，5/01 入庫就 5/31 到期。預設 30 天，跟供應商談好月結 60 就填 60。改了之後只影響未來新進貨，不會回頭改舊應付。
  6. 按 儲存
小提示 如果某個供應商已經不再合作，不要刪掉（會找不到歷史進貨單、歷史應付），把「狀態」改成停用就好。
8進貨單 adminmanagerwarehouse

進貨單是「向供應商買東西」的紀錄。流程：開單 → 等貨到 → 確認入庫（這時候庫存才增加，也才會自動產生應付帳款）。

Step 1 — 開一張進貨單（草稿）

角色：adminmanager

  1. 左側選單點「進貨管理」
  2. 右上角點 + 新增
  3. 選供應商
    選完後系統會用這家供應商身上設定的「付款條件天數」來算之後的應付到期日。
  4. 加進貨項目
    每一列填：商品、數量、單價。可以點「新增一項」加更多商品。
  5. 含稅 / 未稅切換
    如果你填的單價已經是含稅價，勾「含稅」；如果是未稅價，不勾（系統會自動加 5%）。畫面下方會即時顯示「未稅／稅額／含稅總額」三個數字，讓你確認跟對方對得起來。
  6. 備註（可不填）
  7. 按 儲存
    系統會給這張單一個編號，像 PO-20260514-0001。狀態是草稿。還沒影響庫存、也還沒產生應付。

Step 2 — 貨到了，確認入庫

角色：adminmanagerwarehouse

  1. 在進貨單清單找到那張草稿
  2. 點綠色勾勾「確認入庫」
  3. 確認對話框出現後按 確認

狀態變成已入庫，這一刻系統會「一次做三件事」：

  • 每樣商品的「庫存量」會自動加上這張單的數量。
  • 每樣商品的「成本」會被更新為這張單的單價（系統記住最新成本）。
  • 自動產生一張應付帳款（編號 AP-YYYYMMDD-####），狀態 未付，到期日 = 入庫日 + 供應商「付款條件天數」。可以到「應付帳款」頁面去看 / 記付款。

不要的單怎麼處理

只有草稿狀態才能取消。點紅色叉叉「取消單據」即可。

不能改 已入庫的進貨單不能再修改也不能取消，因為它已經牽動庫存 + 應付帳款。如果發現有錯，找主管處理 — 通常會用「庫存調整」沖掉錯誤的數量，並在備註寫明原因。
9客戶 adminmanagersales

記錄你賣給誰。建立銷貨單之前要先有客戶。

新增客戶

  1. 左側選單點「客戶」
  2. 右上角點 + 新增
  3. 填客戶名稱
    必填，名稱不能跟既有的重複。
  4. 聯絡人、電話、Email、地址、統編、備註（看需要填）
  5. 付款條件（天）
    這欄決定之後「銷貨單確認出貨」自動產生的應收帳款「到期日」要算幾天。預設 30。月結 60 就填 60、貨到付款就填 0。改了之後只影響未來新銷貨，不會回頭改舊應收。
  6. 按 儲存
10銷貨單 adminmanagersales

銷貨單是「賣東西給客戶」的紀錄。流程跟進貨單一樣分兩階段：開單 → 確認出貨（這時候庫存才減少，也才會自動產生應收帳款）。

Step 1 — 開一張銷貨單（草稿）

  1. 左側選單點「銷貨管理」
  2. 右上角點 + 新增
  3. 選客戶
    選完後系統會用這個客戶身上設定的「付款條件天數」來算之後的應收到期日。
  4. 選業務員
    「業務員」是業績歸屬，跟「實際在輸入這張單的人」可能不同（例如主管幫業務 A 鍵單，業務員選 A）。如果你自己就是業務員角色，系統會預設選你自己。
  5. 加銷貨項目
    選商品時，單價會自動帶入商品本身的售價，可以再改。如果數量超過目前庫存，數量欄會變紅提醒你 — 但這時還沒扣庫存，存草稿不會被擋。
  6. 含稅 / 未稅切換
    跟進貨單一樣 — 看你給客戶的報價是含稅還是未稅就勾對應的。下方會即時顯示「未稅／稅額／含稅總額」。
  7. 按 儲存
    系統會給這張單一個編號，像 SO-20260514-0001。狀態是草稿。還沒扣庫存、也還沒產生應收。

Step 2 — 出貨，確認

  1. 在銷貨單清單找到那張草稿
  2. 點綠色勾勾「確認出貨」
  3. 確認對話框出現後按 確認

狀態變成已出貨。這一刻系統會「一次做兩件事」：

  • 先檢查每個商品庫存是否夠 — 任何一個不夠都會整張單都不能出，並顯示哪些差多少。都夠的話，把每個商品的庫存量扣掉。
  • 自動產生一張應收帳款（編號 AR-YYYYMMDD-####），狀態 未收，到期日 = 出貨日 + 客戶「付款條件天數」。可以到「應收帳款」頁面去看 / 記收款。
庫存不夠怎麼辦 錯誤訊息會告訴你哪個商品差多少。先去開一張進貨單把貨補進來、確認入庫，再回來確認這張銷貨單。
業務員 vs 建單者 系統會分別記下「業績算誰的」（業務員）和「實際是誰按鍵儲存的」（建單者）。所以主管代鍵單給業務，業績還是算給業務。
11應收帳款（賣出去的錢還沒收） adminmanagersales

「應收帳款」是客戶欠你的錢的清單。它不是手動建的 — 每次「銷貨單確認出貨」系統就會自動生一張。你在這頁要做的事就是：看清單、記客戶付了多少、看誰逾期了。

畫面有什麼

  • 清單：應收編號（AR-YYYYMMDD-####）、對應的銷貨單編號、客戶、未稅金額、稅額、含稅總額、未收餘額、狀態、到期日。
  • 過濾器：搜尋（按應收編號）、客戶、狀態、「只看逾期」開關。
  • 頂部統計：當前清單裡所有應收的「總金額」與「未收餘額」。打開「只看逾期」就會變成逾期金額總計。
  • 狀態：未收 / 部分收款 / 已收齊 / 作廢。逾期會多一個紅色 逾期 chip。

記一筆收款（客戶付錢進來時）

角色：adminmanagersales

  1. 在清單找到那筆應收
    可以用客戶過濾或搜尋編號。
  2. 點眼睛圖示「檢視」
    會跳出明細：客戶、對應銷貨單、開立時間、到期日、未稅/稅額/總額、已收/未收餘額、以及這張應收的所有收款紀錄。
  3. 點「記錄收款」按鈕
    如果這張已經 已收齊 或 作廢，按鈕不會出現。
  4. 填收款資訊
    • 金額：預設帶入未收餘額，可以改成更少（分次收）。不能超過餘額。
    • 方式：現金 / 銀行轉帳 / 支票 / 其他。
    • 收款日期：預設今天，可改。
    • 參考號（選填）：例如轉帳交易序號、支票號碼。
    • 備註（選填）：對帳用。
  5. 按 儲存
    收款編號自動產生 REC-YYYYMMDD-####。應收的「未收餘額」會自動更新；如果剛好收齊，狀態自動跳 已收齊；中途就是 部分收款。

不小心記錯了 — 作廢一筆收款

在明細視窗的「收款紀錄」表格，每一筆右邊有個紅色「作廢」按鈕。

  1. 按「作廢」
  2. 填作廢原因
    建議寫清楚，例如「客戶要求退回此筆、改用 5/20 那筆」。
  3. 確認
    那筆紀錄不會被刪掉（審計留底），而是被劃掉並標 作廢；應收的未收餘額會回補，狀態也會自動回算（可能從 paid 退回 partial 或 open）。
注意 應收帳款的金額本身不能直接改，也沒有「刪除應收」按鈕。如果應收金額有錯，根本原因通常在原始銷貨單 — 找主管處理。若是收款記錯了，用「作廢」沖掉、重記一筆才對。
12應付帳款（買進來的錢還沒付） adminmanager

「應付帳款」是你欠供應商的錢的清單，跟「應收」是鏡像關係。每次「進貨單確認入庫」系統就會自動生一張。

畫面有什麼

  • 清單：應付編號（AP-YYYYMMDD-####）、對應的進貨單編號、供應商、未稅/稅額/含稅總額、未付餘額、狀態、到期日。
  • 過濾器：搜尋（按應付編號）、供應商、狀態、「只看逾期」開關。
  • 頂部統計：清單上應付的總金額與未付餘額。

記一筆付款（你付錢給供應商時）

角色：adminmanager（業務不行）

  1. 在清單找到那筆應付
  2. 點眼睛圖示「檢視」
    明細：供應商、對應進貨單、開立時間、到期日、金額拆解、已付/未付餘額、付款紀錄。
  3. 點「記錄付款」按鈕
  4. 填付款資訊
    金額（不能超過餘額、預設帶入未付餘額）、方式（現金/轉帳/支票/其他）、付款日期、參考號（轉帳序號、支票號碼）、備註。
  5. 按 儲存
    付款編號自動產生 PAY-YYYYMMDD-####。應付未付餘額更新，狀態自動跳。

作廢付款

跟應收一樣 — 在付款紀錄列按「作廢」並填原因。原紀錄會被劃掉留底，未付餘額回補，狀態自動回算。

分次付 / 合單付 系統「每張進貨單對到一張應付」，所以如果一張匯款想沖掉好幾張應付，需要一張一張分別記。建議在「參考號」欄位都填同一個轉帳序號方便事後對帳。
13帳齡分析（誰拖了多久） adminmanagersaleswarehouse

把所有「還沒付清的應收 / 應付」按「離到期日多久」分成五桶，幫你一眼看出哪些客戶最該催、哪些供應商欠最久。

怎麼看

  1. 左側選單「會計系統」→「帳齡分析」
  2. 選分頁：「應收帳齡」（客戶欠你）或「應付帳齡」（你欠供應商）。
  3. 選「截止日」：預設今天。改成 5/31 就是「假設今天是 5/31 的話」的帳齡。

五個桶代表什麼

未到期
到期日還沒到 — 正常範圍，不用催。
1-30 天
過了到期日 1 到 30 天 — 開始該提醒。
31-60 天
過了到期日 31 到 60 天 — 要積極催了。
61-90 天
過了 61 到 90 天 — 信用拉警報。
90 天以上
超過 90 天 — 通常已經是呆帳風險。

每一列是一個客戶（或供應商），把他名下所有未付清的應收/應付的「未收/未付餘額」依到期日分到對應的桶。表格按金額大小排序，最該關心的會排在最上面。

已收齊 / 已付清的不會出現 帳齡只看「還沒結清」的單。一旦付清，那筆就從帳齡裡消失了。
14查庫存 adminmanagersaleswarehouse

隨時查現在每個商品還剩多少。

  1. 左側選單點「庫存查詢」
  2. 看清單
    每一列：商品編號、名稱、分類、目前庫存、低庫存閾值、是否警示。
  3. 勾「只看低庫存」
    只看那些已經低於閾值的商品，方便決定要進什麼貨。
  4. 搜尋框
    可以按商品編號或名稱即時過濾。
小提示 低庫存的列會用淡黃色背景跟「低庫存」紅色 chip 標出來，一眼就看得到。
15庫存調整（盤點） adminmanagerwarehouse

實際庫存跟系統數字對不上的時候，用這個功能補正。例如盤點發現多／少了、商品報廢、其他原因。

新增一筆調整

  1. 左側選單點「庫存調整」
  2. 右上角點 + 新增
  3. 選商品
    下拉清單或搜尋商品編號／名稱。選好後系統會顯示「目前庫存」。
  4. 輸入變動量
    正數＝庫存增加（盤盈），負數＝庫存減少（盤虧／報廢）。例如盤點實際多 3 個就填 3；少了 5 個就填 -5。
  5. 選原因
    四選一：盤盈、盤虧、報廢、其他。系統只把這當「標籤」，真正影響庫存的是你填的變動量數字。
  6. 備註（可選）
    建議寫清楚為什麼要調，方便日後查證。例如「2026/05/01 盤點：A 倉架上實有 8 個」。
  7. 確認預覽，按 儲存
    儲存前畫面會顯示「目前庫存 → 調整後庫存」。如果調整後會變負數，會用紅字擋下不讓你存。
不要這樣做 調整單一旦存檔，不能修改、也不能刪除。這是給審計查證用的紀錄，每一筆都會留下「誰、什麼時候、為什麼調」。發現填錯只能再開一筆相反方向的調整單來抵消，並在備註寫明原因。

查詢過去的調整紀錄

  1. 進「庫存調整」頁就是清單
    每一列：調整單號（ADJ-YYYYMMDD-####）、商品、調整前／變動／調整後、原因、操作人、時間。
  2. 用上方過濾器篩選
    可以按商品、原因、操作人過濾，或用搜尋框找單號。
會影響月報嗎？ 會。在「月報表」每個商品那一列，盤點異動欄會顯示當月所有調整的有號淨變動（例如盤盈 +3、盤虧 −5，淨值 −2）；期末庫存的計算也會把它算進去。
16月報表 adminmanagersaleswarehouse

看某個月每個商品的進銷情況。也可以匯出 Excel 拿給老闆看。

  1. 左側選單點「報表」
  2. 選年份、月份
  3. 看內容
    每個商品一列：期初庫存、本月進貨、本月銷貨、盤點異動、期末庫存、進貨金額、銷貨金額。最上方有「進貨合計」「銷貨合計」總數。
  4. 匯出 Excel（可選）
    點 匯出 Excel，瀏覽器會下載一個 .xlsx 檔，雙擊就能用 Excel 打開。
怎麼算的 只有「已入庫」的進貨和「已出貨」的銷貨會被算進去。草稿和已取消不會算。盤點異動（見第 12 章）也會列入期末庫存的計算。
17使用者帳號（系統管理員專用） admin

這一章只有 admin 看得到「使用者管理」。其他角色看不到也不能用。

幫同事開新帳號

  1. 左側選單點「使用者管理」
  2. 右上角點 + 新增
  3. 填基本資料
    帳號、姓名、Email、密碼（至少 8 字）。
  4. 選角色
    adminmanagersaleswarehouse 擇一。
  5. 按 儲存

幫人家改密碼

  1. 找到那個人，點鉛筆「編輯」
  2. 新密碼欄輸入新的密碼
    留空表示不改密碼。輸入了就會被替換。
  3. 按 儲存

停用一個人

編輯使用者，把「狀態」改成「停用」儲存。停用後他登不進來，但歷史單據還在。建議用「停用」而不是「刪除」。

注意 你不能刪除自己（避免不小心鎖住系統）。
18遇到問題怎麼辦

我打錯字了，怎麼救？

  • 還是草稿狀態：點鉛筆編輯就能改。
  • 已經已入庫／已出貨：不能改了。找主管處理。
  • 應收 / 應付金額看起來怪怪的：金額是從進貨單 / 銷貨單來的，要從原始那張單查根本原因；單筆收/付款記錯就用「作廢」沖掉、重記。

確認出貨時被擋住，說庫存不夠

系統會告訴你哪幾個商品差多少。先去進貨單那邊建單、確認入庫補貨，再回來確認這張銷貨單。

為什麼我銷貨單按了確認，但「應收帳款」清單沒看到？

  • 檢查銷貨單狀態真的是 已出貨 嗎？只有確認後才會產生應收。
  • 應收清單預設按建立時間倒序，最新的應在最上面。也可以用客戶過濾或搜尋對應的 AR-… 編號。
  • 確認權限：四種角色都能看應收，但 warehouse 看不到「記錄收款」按鈕，這正常。

「付款條件」改了之後，舊的應收應付會跟著改到期日嗎？

不會。到期日是「確認入庫 / 出貨」那一刻算好寫進該張應收應付的，之後就不會變。改客戶/供應商上的天數只影響「之後新產生」的應收應付。

含稅 / 未稅選錯了，現在已經確認了怎麼辦？

已確認的單不能改。如果差異不大就在備註裡寫清楚事後手動對帳；金額影響大的話找主管，可能要建反向單沖銷。下次建單前再確認一次。

客戶要分次付款，怎麼處理？

就在同一張應收上「記錄收款」記幾次。每次填當次實收金額，未收餘額會逐步降到 0。狀態會從 未收 → 部分收款 → 已收齊。應付也是一樣道理。

「業務員」跟「建單者」有什麼差別？

  • 業務員 = 業績算誰的（用來算每月每人賣多少）。
  • 建單者 = 實際按下「儲存」這個動作的人。系統會自動記錄為當下登入的你，不用選。
  • 例：主管幫業務 A 鍵了一張單 — 業務員選 A，建單者自動是主管。月底算業績算給 A。

我看不到某個按鈕

你的角色沒有那個權限。例如sales 角色不能修改商品。如果這影響你做事，找主管把你的角色調整成需要的等級。

怎麼切換語言

右上角的「中文 ▾」可以切到 English。下次再開系統會記住你的選擇。

我登不進去

  • 確認帳號密碼有沒有打錯（大小寫要對）。
  • 確認帳號沒被「停用」。
  • 找系統管理員確認 / 重設密碼。

畫面看起來怪怪的

按 Ctrl + F5 重新整理（同時清快取）。還是怪的話，登出再登入。
